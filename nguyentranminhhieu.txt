câu 4:Phân biệt git merge và git rebase(giống, khác nhau, khi nào nên sử dụng)

-giống: đều dùng để gộp nhánh 

-khác: + merge: khi gộp 2 nhánh sẽ tạo ra commit merge và gộp 2 commit mới 	       nhất của 2 nhánh với nhau
      + rebase: khi gộp 2 nhánh không tạo ra commit mới nào thay vào đó là                    	        đổi mã hash của commit 
-sử dụng merge khi gộp 2 nhánh của 1 dự án. sử dụng rebase sau khi fetch để push code lên remote

câu 17: Phân biệt HEAD và BRANCH

HEAD  : có thể trỏ đến BRANCH hoặc trực tiếp đến COMMIT
BRANCH: chỉ có thể trỏ đến commit, chỉ nhảy lên khi HEAD trỏ đến và tạo 	commit mới 

câu 19: Liệt kê các trạng thái của file trong git, giải thích từng trạng thái

có 4 trạng thái:
	+ untracked : file mới được tạo
	+ staged    : khi git add (sẵn sàng để commit)
	+ unmodifde : file ở trạng thái đã commit 
	+ modifide  : file đã được commit và đã chỉnh sửa
