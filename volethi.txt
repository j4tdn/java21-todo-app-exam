2. Viết lệnh thực hiện gộp nhánh bug vào nhánh feature sử dụng git merge

git checkout feature
git merge bug

3. Viết lệnh thực hiện gộp nhánh feature vào bug sử dụng git rebase

git checkout bug
git rebase feature
 

4. Phân biệt git merge và git rebase(giống, khác nhau, khi nào nên sử dụng)

Giống: - Tạo thành 1 nhánh thẳng hàng
       - Đều là lệnh gộp nhánh

              + chỉ fix conflicts 1 lần

      -Rebase:+ không thay đổi commit(số lượng)
              + fix conflicts nhiều lần

5. Phân biệt git fetch và git clone

git fetch: cập nhật các commit mới nhất trên remote
git clone: cập nhật toàn bộ dự án trên remote về 

6. Phân biệt git fetch và git pull

git fetch: cập nhật các commit mới nhất trên remote
git pull = git fetch + git merge

7. Phân biệt git pull và git pull --rebase

git pull là git fetch + git merge
git pull --rebase là git fetch + git rebase

8. Git log dùng để làm gì

git log dùng để show chi tiết các commit, ai là người viết và thời gian

9. Git status dùng để làm gì

git status dùng để xem trạng thái các file chúng ta đang làm việc

10. Git add dùng để làm gì

git add dùng để chuyển trạng thái file ở untracked sang stage để sẵn sàng commit

11. Git commit dùng để làm gì

git commit dùng để tạo 1 commit mới và chuyển file từ stage sang unmodified

12. Mô tả lệnh để tạo branch ở local

git branch new_branch
git branch new_branch old_branch
git branch new_branch hash

13. Mô tả lệnh để xóa branch ở local

git branch -D name_branch

14. Mô tả lệnh để xóa branch ở remote

git branch -d name_branch

15. Nhánh trung gian là gì

Nhánh trung gian là 1 nhánh liên kết giữa remote và local

16. Detach HEAD là gì

Detach HEAD là trạng thái commit ko được nối vào nhánh nào 

17. Phân biệt HEAD và BRANCH

HEAD là working directory và được trỏ đến các commit(detach head) và branch
Branch là các nhánh được tạo ra, đại diện cho các version

18. Có mấy cách để xóa commit, mô tả và cho ví dụ

git reset --hard HEAD~n
n: số lượng commit tính từ commit hiện tại
git reset --hard HEAD~1

19. Liệt kê các trạng thái của file trong git, giải thích từng trạng thái

untracked: trạng thái mới tạo file
modified: trạng thái đang sửa file
unmodified: trạng thái ko thể sửa file
stage: trạng thái chuẩn bị commit
